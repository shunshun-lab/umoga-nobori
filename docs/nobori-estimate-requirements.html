5. 検収条件

検収は本番環境（クライアント管理の Vercel + Shopify ストア）を対象に実施します。 以下の全「必須」項目をクリアした時点で検収完了とします。検収期間は納品報告から 10営業日 とし、 指摘事項は 2営業日以内 に修正対応します。

5.1 見積もり機能

  • AC-01 定型サイズ（レギュラー等）を選択し、単価・合計・内訳が 1 秒以内に更新される
  • AC-02 カスタムサイズ入力時、10cm 単位の切上げ計算が価格に反映される
  • AC-03 枚数を変更すると数量割引が正しい率で適用される（Sheets 同期後のルールと一致）
  • AC-04 付属品の個数変更が見積もり内訳に明示的に反映される
  • AC-05 規定外サイズ（150cm×3000cm 超等）で「見積もり依頼」フローに切り替わる

5.2 カート・決済フロー

  • AC-06 異なる仕様の商品を 2 件以上カートに追加し、ページリロード後もカートが保持される
  • AC-07 お急ぎ便選択時、商品合計の surcharge（デフォルト 20%）が別 line item として請求書に表示される
  • AC-08 氏名・電話・メール入力後、「Shopify 決済へ進む」で請求書ページにリダイレクトされる
  • AC-09 Shopify 請求書に priceOverride による正しい金額が表示される（フロント計算と一致）
  • AC-10 決済完了後、Shopify 管理画面「注文」にレコードが作成される
  • AC-11 決済完了後、アプリ側カートがクリアされる

5.3 Shopify 注文データの完全性

テスト注文 1 件について、以下の line item properties が全て記録されていること：

  • AC-13 同期後、変更した割引ルールが見積もりに即時反映される

6. 見積もり（松竹梅プラン）

6.1 単価の考え方と根拠

本案件では、金額の透明性のため工数（人日）を先に積み上げ、単価を掛けてプラン総額を算出します。 単価には ¥50,000 / 人日（本見積の適用単価）と、市場参考の ¥80,000 / 人日 の2段階を併記し、 なぜ本案件では ¥50,000 が妥当かを以下に整理します。

¥80,000 / 人日が妥当とされる場合（市場参考単価）

根拠説明
フルスタック + Shopify 専門性 React フロント、Vercel Functions、Shopify Admin API（Draft Order / priceOverride）、Google Sheets / Firebase 連携を一人で担う場合、一般的なフリーランス相場は ¥70,000〜¥100,000 / 人日（税抜）に収まる。
本番障害リスク 決済・注文データの不整合は売上直結のため、本番検証・切り戻し設計に上乗せコストが発生する。未知のコードベースであれば探索工数（1〜3人日）が別途必要。
ゼロからの新規開発 要件定義・設計・実装・テストを一気通貫で請け負う場合、PM・レビュー・ドキュメント作成を含め ¥80,000 は妥当な基準単価。
法人・エージェンシー経由 社内マージン・保険・契約事務を含む場合、エンジニア単価が ¥80,000〜¥120,000 に相当することが多い。

¥50,000 / 人日を本案件に適用する理由（本見積の採用単価）

根拠本案件への当てはめ
実装進捗 約85% 見積もり UI・価格計算・Draft Order 決済・管理画面のコアは実装済み。残作業は「仕上げ・本番化・移譲」が中心で、新規設計工数が小さい。
コードベースの継続性 同一開発者・同一リポジトリでの継続開発のため、キャッチアップ・仕様把握コスト（通常 1〜3人日）が不要。これを単価に反映するのが合理的。
スコープの明確さ 未完了項目（Firebase、E2E、権限移譲等）は TODO として列挙済み。不確実性プレミアムを抑えられる。
作業内容の性質 残作業の多くは設定・検証・ドキュメント整備であり、ゼロからのアプリ設計より単価感が下がる領域。
継続案件としての調整 UMOGA様との既存関係・デモ環境の引き継ぎを前提とした継続開発単価として ¥50,000 を提示。新規受託の初回案件であれば ¥80,000 相当が妥当。
結論（推奨）： 本案件は ¥50,000 / 人日 を適用単価とするのが妥当です。 ¥80,000 は「新規受託・未知のコードベース・PM込み」の参考単価として併記します。 梅プラン ¥700,000 = 14人日 × ¥50,000 は、残作業の最小スコープと整合します。 同一内容を ¥80,000 で積むと 8.75人日相当となり、本番 E2E・権限移譲・Firebase 修復を品質担保した場合の現実工数を下回ります。

7. 前提条件・除外事項

7.1 前提条件

  • クライアントより Shopify ストアの管理者権限、Google アカウント、ドメイン（必要な場合）を提供いただく
  • 本番テスト注文の決済はテストモード or 少額実決済で実施（決済手数料はクライアント負担）
  • 価格マスタ・割引ルールのビジネスロジックは現行 Sheets の内容を正とする
  • デザイン素材（バナー画像・テンプレート PDF）はクライアント提供 or 既存デモ素材を流用
  • Shopify API バージョンは 2025-01 以上を使用（priceOverride 必須）

7.2 除外事項（本見積に含まない）

  • Shopify 月額プラン・決済手数料・ドメイン費用
  • 新規デザイン制作（ロゴ・バナーの撮影・イラスト制作）
  • のぼり製造・物流オペレーションの設計
  • SEO / 広告運用 / コンテンツマーケティング
  • Firebase Blaze プラン等、無料枠を超えるインフラ従量課金
  • 24時間365日の運用監視・SLA 付き保守（別途保守契約）
