## 5. LINE公式アカウントの運用特徴

### 5.1 導線設計

```
LINE友だち追加
  ├─ トーク画面（リッチメニュー）
  │   ├─ ポイントアプリ（civicship.app）へ誘導
  │   ├─ mobi（地域交通）アプリDL + 紹介コード
  │   ├─ 宿泊施設の紹介・予約
  │   └─ イベント情報の閲覧
  └─ LINE VOOM（タイムライン）
      ├─ イベント告知（カルーセル画像）
      ├─ YouTube番組の宣伝
      └─ クエスト（お手伝い）の募集
```

### 5.2 チャネル間の連携

| チャネル | 役割 |
|----------|------|
| **LINE公式アカウント** | メイン接点。リッチメニューから各サービスへの導線ハブ |
| **civicship.app** | ポイント管理・クエスト予約・体験予約の実行基盤 |
| **mobi アプリ** | 地域交通サービス。紹介コードで連携 |
| **YouTube** | 毎週のライブ番組で認知拡大・関係人口の維持 |
| **LINE VOOM** | イベント告知・番組宣伝のタイムライン配信 |

### 5.3 ターゲティング

- **関係人口（域外ファン）**: 高速バス・宿泊の導線を重視
- **来訪者**: mobi紹介コードで移動手段を提供
- **リピーター**: 毎週のYouTube番組でエンゲージメント維持
- **お手伝い希望者**: クエスト告知で労働力確保

## 6. Webアプリ（civicship.app）ユーザー画面

LINEログイン後にアクセスできるWebアプリのマイページ・設定画面を確認。

### 6.1 マイページ

- **ユーザー名表示**: 本名（例: 髙木俊輔）
- **アバター**: 名前の頭文字を円形アイコンで表示
- **保有ポイント**: `0pt` と表示（ポイント詳細ページへの遷移矢印あり）
- **これまでの関わり**: 参加したクエスト/体験の履歴セクション
  - 空の場合「地域の素敵な人との関わりを探してみましょう」+ 「関わりを探す」CTAボタン
  - 「すべて見る」リンクで履歴一覧ページへ遷移可能
- **ボトムナビ**: 「見つける」「マイページ」の2タブ構成

### 6.2 設定ページ

| セクション | 項目 | 詳細 |
|-----------|------|------|
| **アカウント** | 共創ID | `did:prism:a0bcc4...dece2`（DID形式の分散型ID） |
| | LINEアカウント | 連携済み |
| | 電話番号 | 連携済み |
| **設定** | プロフィール | 「編集」ボタンで編集画面へ |
| **利用にあたっての約束** | 利用規約 | 閲覧可能 |
| | プライバシーポリシー | 閲覧可能 |
| **ご注意して操作して下さい** | サインアウト | ログアウト機能 |
| **お知らせ** | リリース情報 | アプリの更新情報 |

### 6.3 プロフィール編集ページ

| フィールド | 必須 | 備考 |
|-----------|------|------|
| **プロフィール画像** | 必須 | 「画像を選択」ボタンでアップロード |
| **本名** | 必須 | 「どなたか分からなくなるため、必ず本名をご入力ください」の注釈あり |
| **自己紹介** | 任意 | テキストエリア（プレースホルダー「自己紹介を入力しましょう」） |
| **SNSリンク** | 任意 | Facebook / Instagram / X（旧Twitter）の3つ |
| **保存ボタン** | — | 青色のCTAボタン |

### 6.4 すべての関わり（履歴一覧）ページ

- **タブ切り替え**: 「今後の予定」/「過去」の2タブ
- **検索機能**: 「キーワードで検索」のテキスト入力
- 新規ユーザーの場合、両タブとも空表示

### 6.5 技術的な注目点

- **DID（分散型ID）の採用**: `did:prism` スキームを使用。Cardano系のAtala PRISM（現Identus）ベースのDID
- **LINE + 電話番号の二重連携**: ソーシャルログインに加え電話番号認証も実施
- **本名必須ポリシー**: 匿名利用を排除し、地域コミュニティとしての信頼性を担保

## 7. 気になった点 / 改善候補

1. **リッチメニューの情報密度が高い** — ポイント・宿泊・mobi・イベントが1画面に詰まっており、初見ユーザーには分かりにくい可能性
2. **mobi紹介コードの訴求が弱い** — QRコード2枚 + テキストのみで、メリット（無料特典等）の説明がない
3. **宿泊施設が2件のみ** — 選択肢が少なく、ポイント利用との連携（ポイントで宿泊割引等）が不明
4. **VOOM投稿のCTA不足** — イベント告知はあるが、「予約はこちら」等の直接リンクが見当たらない
5. **YouTube番組とポイントの連動なし** — 視聴ポイント付与等でcivicship.appへの流入を増やせる余地あり
6. **ポイント説明パネルの「遊体験※」** — 「遊体験」の表記に※マークがあるが注釈が見当たらない（誤字の可能性）

### Webアプリ（civicship.app）側
7. **マイページの情報量が少ない** — ポイント残高と履歴のみ。バッジ、ランキング、達成度などのゲーミフィケーション要素がない
8. **プロフィール画像が必須だが初期値が空** — 登録直後にアイコンが頭文字のみで表示され、コミュニティ感が薄い
9. **DIDの表示が技術寄り** — `did:prism:a0bcc4...dece2` がそのまま表示されており、一般ユーザーには意味不明。「あなたのデジタル証明書」等の分かりやすいラベルが欲しい
10. **「ご注意して操作して下さい」の表記** — 「ご注意して」は日本語として不自然。「ご注意ください」または「慎重に操作してください」が適切
11. **履歴ページの空状態が寂しい** — 初回ユーザーに対して「おすすめのクエスト」等のサジェストがあると良い

## 8. Webサイトとの比較

| 観点 | Webサイト（civicship.app） | LINE公式アカウント |
|------|---------------------------|-------------------|
| **主な役割** | クエスト/体験の検索・予約・ポイント管理 | 告知・集客・外部サービスへの導線 |
| **認証** | LINEログイン | LINE友だち追加（認証不要で閲覧可） |
| **コンテンツ更新** | クエスト/体験の追加・更新 | VOOM投稿・リッチメニュー更新 |
| **強み** | 予約・ポイント管理の実行機能 | プッシュ通知・日常的な接点 |
| **弱み** | 能動的にアクセスが必要 | 予約等のトランザクション機能なし |
