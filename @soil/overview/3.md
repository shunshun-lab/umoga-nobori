# 今後の開発・改善タスク一覧

本プロジェクトのさらなる発展と品質向上のための推奨タスクを以下のカテゴリ別にまとめました。

## 1. UI/UX の改善
- **デザイン入稿データのリアルタイム検証**:
  - アップロードされた画像の解像度やアスペクト比を自動チェックし、印刷に適さない場合に警告を表示。
- **デザインプレビューの高度化**:
  - のぼり旗が実際に設置された状態のシミュレーション（背景合成）機能。
- **モバイル操作性の追求**:
  - モバイル版での見積もり詳細画面をスワイプで閉じられるように改善。
  - 選択項目の変更時に微細なマイクロインタラクション（アニメーション）を追加。
- **ダークモードの完全対応**:
  - 現在のライトモードベースのデザインを、ユーザーのシステム設定に応じて最適化。

## 2. 機能・ロジックの拡張
- **デザイン制作依頼フォームの構造化**:
  - 現在の自由記述式から、色指定、ロゴ有無、雰囲気の選択肢（ポップ、和風、モダン等）などを選べる構造化フォームへ改善。
- **商品ラインナップの拡充**:
  - 横断幕、タペストリー、ミニのぼりなど、新たなカテゴリの価格計算ロジックとUIの追加。
- **外部ストレージ連携の強化**:
  - 巨大なファイルをブラウザからS3やGoogle Driveへ直接アップロードする機能。
- **多言語・多通貨対応**:
  - 将来的な海外展開を見据えた i18next 等の導入。

## 3. インフラ・外部連携
- **Shopify Webhook による注文同期の完全自動化**:
  - 決済完了時に自動的に注文情報をバックエンドで処理し、ステータスを更新。
- **Google Sheets への注文内容書き込み**:
  - 新規注文が入った際、管理用のスプレッドシートに自動で詳細行を追加。
- **通知システムの構築**:
  - デザイン入稿忘れや、制作進行ステータスの変更をメールやLINEでユーザーに通知。

## 4. 管理・メンテナンス
- **パラメータ管理の柔軟化**:
  - Google Sheets だけでなく、管理画面から直接ベース価格や割引率をオーバーライドできる機能。
- **エラーハンドリングとロギング**:
  - APIエラーやハイドレーションエラー等の監視を Sentry 等で強化。
- **自動テストの導入**:
  - 計算ロジックだけでなく、React Testing Library や Playwright によるUIの自動回帰テスト。
