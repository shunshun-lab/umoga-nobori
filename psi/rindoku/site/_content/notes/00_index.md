# 00_index — 全体像

> **書名**: Computational Science and Engineering (Gilbert Strang)
> **テーマ**: 現実の問題を「モデル化 → 離散化 → 解法」で解く数学的道具箱

---

## 本全体の要約

現象を方程式（行列・微分方程式）に翻訳し、離散化して計算で解く——この一連の流れを、線形代数・フーリエ・複素解析・最適化という道具で統一的に扱う本。「解析（アイデア）」と「計算（アルゴリズム）」を分離せず、構造を見抜いて正しい解法を選ぶ力を鍛える。

## 重要な論点

1. **アルゴリズム選択 > 計算資源**: 力任せに解くより、構造に合った解法を選ぶ方が投資対効果が高い
2. **精度と安定性は別物**: 良い近似でも離散化が不安定なら壊れる（Ch6 の |G|>1）
3. **特異点が限界を決める**: フーリエの Gibbs（Ch4）も複素の収束半径（Ch5）も、不連続/特異点の位置が性能上限
4. **最小化は方程式の生成装置**: 目的関数を置くだけで、解くべき方程式（KKT, Euler-Lagrange）が自然に出る（Ch8）
5. **局所ルールの積み上げ = 全体方程式**: ばね・回路・グラフ——A と C を見つければ K が組める（Ch2）

## 自分にとって重要なポイント

- **問題分解の型**: 「モデル化（A, C を決める）→ 離散化（差分/FEM）→ 解法（直接/反復）」で詰まり箇所を特定する
- **失敗の予測**: 不連続 → Gibbs、特異点が近い → 収束遅い、帯幅が広い → フィルイン爆発、と事前に見積もれる
- **デバッグの言語化**: 「解が変」を「境界条件の誤り」「零空間の存在」「|G|>1 の不安定」など構造的原因に切り分ける
- **説明責任**: 近似の前提（線形化・定係数化・境界の選択）を「便利な嘘」として位置づけ、適用範囲を言語化できる
- **スケール感覚**: 2D自然順序LUは N^4 級、という粗い見積もりで早期に手法を切り替える判断ができる
