## PS 7.1 — 帯幅とフィルインの関係

**問題**: N×N 格子（N=4）の2Dポアソン方程式を自然順序（行優先）で番号付けしたとき、行列の帯幅 w と LU 分解のフィルインを見積もれ。

**回答**:

自然順序（行ごとに左→右、下→上）: 未知数 n = N² = 16。

5点ステンシルにより、隣接格子点との結合がある。同じ行の隣 → 帯幅内。隣の行の点 → N だけ離れた位置。

**帯幅**: w = N = 4（Kの主対角から最も遠い非零要素までの距離）。

**LU のフィルイン**: 帯幅 w の帯行列を LU 分解すると、L と U の帯内がすべて埋まる。

- 保存量（メモリ）: O(nw) = O(N² · N) = **O(N³)**
- 計算量: O(nw²) = O(N² · N²) = **O(N⁴)**

N = 100 なら n = 10⁴、計算量 ≈ 10⁸。N = 1000 なら n = 10⁶、計算量 ≈ **10¹²**（実用不可能）。

**核心**: 2D問題の「自然順序LUは N⁴」。これが反復法への動機。

## PS 7.1 — ネステッドディセクションによる改善

**問題**: 同じ N×N 格子で、ネステッドディセクション（入れ子分割）を使うとフィルインがどう改善するか。

**回答**:

**アイデア**: 格子を2分する「セパレータ」を見つけ、セパレータの変数を最後に番号付けする。再帰的に適用。

**2D 格子の結果**:
- セパレータサイズ: O(N)
- 再帰の深さ: O(log N)
- フィルイン: O(N² log N)（自然順序の O(N³) から改善）
- 計算量: **O(N³)**（自然順序の O(N⁴) から改善）

| 順序付け | メモリ | 計算量 |
|---|---|---|
| 自然順序（行優先） | O(N³) | O(N⁴) |
| ネステッドディセクション | O(N² log N) | O(N³) |
| 反復法（CG + 前処理） | O(N²) | O(N² √κ) |
| マルチグリッド | O(N²) | **O(N²)** |

**核心**: 順序付けは「計算しない」が、計算量を劇的に変える。アルゴリズム選択 > 計算資源の典型例。

## PS 7.2 — ヤコビ法と収束率

**問題**: Ku = f（K₄ = tridiag[-1,2,-1]、n=4）にヤコビ法を適用し、収束率を求めよ。

**回答**:

K = D − L − U（D=対角、L=下三角、U=上三角）。

ヤコビ法: u^{k+1} = D⁻¹(L + U)u^k + D⁻¹f。

反復行列 J = D⁻¹(L + U) = I − D⁻¹K。D = 2I なので J = I − K/2。

J の固有値: μⱼ = 1 − λⱼ(K)/2 = 1 − (2 − 2cos(jπ/5))/2 = cos(jπ/5)。

スペクトル半径: ρ(J) = max|μⱼ| = cos(π/5) ≈ 0.809。

**収束率**: 1反復で誤差が 0.809 倍。p桁の精度に必要な反復数 ≈ p / log₁₀(1/0.809) ≈ 10.9p。

n を大きくすると ρ(J) = cos(π/(n+1)) ≈ 1 − π²/(2n²) → 1。**収束が遅くなる**（反復数 ∝ n²）。

**核心**: ヤコビ法は簡単だが大規模で遅い。CG やマルチグリッドが必要になる理由。

## PS 7.4 — 共役勾配法（CG）の収束

**問題**: SPD 行列 K に対する CG 法の収束率が条件数 κ = λ_max/λ_min に依存することを説明せよ。

**回答**:

CG の誤差（エネルギーノルム ‖e‖_K）:

$$\frac{\|e_k\|_K}{\|e_0\|_K} \leq 2\left(\frac{\sqrt{\kappa}-1}{\sqrt{\kappa}+1}\right)^k$$

K₄（n次）の条件数: κ ≈ 4n²/π²（Ch1 の結果）。

$$\frac{\sqrt{\kappa}-1}{\sqrt{\kappa}+1} \approx 1 - \frac{2}{\sqrt{\kappa}} \approx 1 - \frac{\pi}{n}$$

p桁の精度に必要な反復数 ≈ (n/π)·p·ln 10 ≈ **O(n)**。

**ヤコビ法との比較**: ヤコビは O(n²) 反復、CG は O(n) 反復。

**前処理付き CG**: 前処理行列 M ≈ K で M⁻¹K の条件数を κ' << κ にすれば、反復数 ∝ √κ' にできる。

**核心**: CG は SPD 大規模問題の標準解法。前処理の質が実用性能を決める。

## PS 7.3 — マルチグリッドの考え方

**問題**: ヤコビ法が高周波誤差を速く減衰させるが低周波誤差が残る理由を説明し、マルチグリッドがこれを解決する仕組みを述べよ。

**回答**:

**ヤコビ法の性質**: 反復行列 J の固有値 μⱼ = cos(jπ/(n+1))。
- 高周波（j ≈ n）: μⱼ ≈ −1 → |μⱼ| は大きいが振動的に減衰
- 低周波（j ≈ 1）: μⱼ ≈ 1 − π²/(2n²) → |μⱼ| ≈ 1 → **ほとんど減衰しない**

ヤコビを数回実行すると高周波は消えるが、低周波の誤差がそのまま残る。

**マルチグリッドの解決策**:
1. **Smoothing**: 細かい格子でヤコビ数回 → 高周波誤差を除去
2. **Restriction**: 残った低周波誤差を粗い格子に移す（制限演算子）
3. **Coarse grid solve**: 粗い格子で低周波誤差を解く（低周波は粗い格子の高周波！）
4. **Prolongation**: 粗い格子の修正を細かい格子に戻す（補間演算子）
5. **再帰**: 粗い格子での解法にも同じ手順を適用（V-cycle / W-cycle）

**結果**: 各レベルの仕事量は O(n)。log n レベルで合計 O(n log n)、実用的にはほぼ **O(n)**。

| 手法 | 反復数 | 1反復のコスト | 合計 |
|---|---|---|---|
| ヤコビ | O(n²) | O(n) | O(n³) |
| CG | O(n) | O(n) | O(n²) |
| CG + 良い前処理 | O(√κ') | O(n) | O(n√κ') |
| マルチグリッド | O(1) | O(n) | **O(n)** |

**核心**: マルチグリッドは「全周波数を同時に効率よく処理する」唯一の O(n) 手法。大規模 PDE の本命。
